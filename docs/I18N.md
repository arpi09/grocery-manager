# Internationalization (i18n)

Home Pantry uses a lightweight custom i18n layer in `src/lib/i18n/`.

## Locales

| Code | Language | Role |
|------|----------|------|
| `sv` | Svenska | **Default** — used when no preference is stored |
| `en` | English | Secondary language |

## How locale is chosen

1. **Cookie** `home-pantry-locale` (set when the user switches language)
2. **`Accept-Language`** header (only `sv` / `en`, first visit without cookie)
3. **Fallback** `sv`

Client-side, the choice is also mirrored in **`localStorage`** under `home-pantry-locale`.

## Switching language

Use the **SV | EN** control in:

- **Settings → App → Språk / Language**
- **Login / Register** (top-right on auth pages)

No page reload is required; UI strings update reactively.

## Adding or changing strings

1. Add the key to **`src/lib/i18n/locales/sv.json`** (source of truth).
2. Add the same key to **`src/lib/i18n/locales/en.json`**.
3. Use in Svelte:

```svelte
<script lang="ts">
  import { t } from '$lib/i18n';

  const title = $derived(t('settings.title'));
</script>
```

4. For reactive text inside markup, wrap with `$derived(t('some.key'))` or call `t()` in `$derived.by(...)`.

### Interpolation

```ts
t('dashboard.totalTracked', { count: 12 });
```

### Plural (simple one/other)

```json
"itemCount": "{count} {count, plural, one {vara} other {varor}}"
```

### Missing English keys

If a key exists in `sv.json` but not `en.json`, **Swedish is shown** for English users until translated.

## Server-side strings

For SSR or form actions, import `translate` and pass an explicit locale from `event.locals.locale` or layout data:

```ts
import { translate } from '$lib/i18n/messages';

translate(locale, 'common.errorGeneric');
```

## Navigation labels

Nav items in `src/lib/navigation/nav-config.ts` use `labelKey` (e.g. `nav.home`) instead of hard-coded labels. `MainNav` resolves them with `t()`.

## E2E tests

Default locale is **Swedish**; Playwright specs use Swedish headings (`Logga in`, `Hem`, …). Prefer **`data-testid`** for language-independent selectors when adding new tests.

## Follow-up migration

Admin, inventory detail, meal plans, and server validation messages can be migrated incrementally by moving strings into the JSON catalogs and replacing inline copy with `t()`.
